## 🔍 问题分析

### 为什么R²仍为负？

**R²为负的含义**：
- R² < 0 表示模型比简单使用平均值预测还差
- 但MAPE很低，说明价格预测相对准确
- **矛盾原因**：R²对异常值敏感，可能存在几个大误差样本

### 可能的原因：

1. **数据量不足**
   - 训练样本：121个 (约4个月数据)
   - 需要：建议至少1年以上数据

2. **特征不够**
   - 当前：15个技术指标
   - 可添加：成交额、外资买卖、融资融券等

3. **模型复杂度**
   - 当前：3层LSTM [64, 64, 32]
   - 可尝试：调整层数和单元数

4. **序列长度**
   - 当前：60天
   - 可尝试：30天、20天

## 🎓 关键学习点

### 1. 数据预处理的重要性
- ✅ 输入和输出必须在相同尺度
- ✅ 归一化后需保存scaler
- ✅ 预测时必须反归一化

### 2. 评估指标的选择
- **R²**: 对异常值敏感，不适合时间序列
- **MAPE**: 相对误差，更适合价格预测
- **方向准确率**: 对交易更有意义

### 3. LSTM的特点
- 需要较多数据才能学习复杂模式
- 对数据质量要求高
- 早停防止过拟合很重要

## 🚀 集成到系统

### 创建预测API
```python
import joblib
from tensorflow.keras.models import load_model

def predict_price(symbol, sequence_data):
    # 加载模型和scaler
    model = load_model(f'models/lstm/{symbol}_model.h5')
    scaler_X = joblib.load(f'data/lstm/{symbol}/{symbol}_scaler_X.pkl')
    scaler_y = joblib.load(f'data/lstm/{symbol}/{symbol}_scaler_y.pkl')
    
    # 归一化输入
    X_scaled = scaler_X.transform(sequence_data)
    
    # 预测
    y_pred_scaled = model.predict(X_scaled.reshape(1, 60, 15))
    
    # 反归一化
    y_pred = scaler_y.inverse_transform(y_pred_scaled)
    
    return y_pred[0][0]
```
