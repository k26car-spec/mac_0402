# 🌐 Web 儀表板使用指南

## 🎉 恭喜！Web 平台已安裝完成

### 📋 可用的指令

```bash
# 1. 啟動 Web 平台（前台模式）
./start_dashboard.sh

# 2. 啟動 Web 平台（背景模式）
./start_dashboard_daemon.sh

# 3. 停止 Web 平台
./stop_dashboard.sh
```

---

##  🚀 快速開始

### 方法 1: 前台啟動（推薦用於測試）

```bash
./start_dashboard.sh
```

您會看到：
```
🚀 啟動 AI 主力監控 Web 平台...

📊 平台網址：
   本機: http://127.0.0.1:5000
   區網: http://192.168.x.x:5000

🛑 停止平台: 按 Ctrl+C
```

然後打開瀏覽器訪問：**http://127.0.0.1:5000**

### 方法 2: 背景啟動（長期使用）

```bash
./start_dashboard_daemon.sh
```

平台會在背景運行，即使關閉終端機也會繼續運行。

## 🎯 Web 平台功能

### 1. **儀表板首頁**

顯示：
- ✅ 系統運行狀態（運行中/已停止）
- 📊 今日警示數量
- 📈 監控股票數量
- ⏰ 交易時間狀態
- 🕐 最後檢查時間

### 2. **控制面板**

四個主要按鈕：
- 🟢 **啟動** - 啟動監控系統
- 🔴 **停止** - 停止監控系統
- 📧 **測試** - 測試 Email 通知
- 🔄 **刷新** - 重新載入所有數據

### 3. **監控清單管理**

- ➕ 添加股票代號
- ❌ 移除不需要的股票
- 📋 查看所有監控中的股票

### 4. **主力警示列表**

顯示：
- 🚨 最近的主力警示記錄
- 📊 信心指數百分比
- 🕐 警示發生時間
- 🎨 顏色標示（紅/黃/綠 = 高/中/低信心度）

## 💡 使用場景

### 場景 1: 盤中監控

```
1. 早上 8:50 - 啟動 Web 平台
   ./start_dashboard.sh

2. 打開瀏覽器訪問 http://127.0.0.1:5000

3. 點擊「啟動」按鈕開始監控

4. 觀察警示列表，有新警示會自動出現

5. 收盤後點擊「停止」按鈕
```

### 場景 2: 長期運行

```
1. 啟動背景模式
   ./start_dashboard_daemon.sh

2. 系統會持續監控

3. 隨時用瀏覽器檢查狀態

4. 偵測到主力時自動發 Email

5. 不需要時執行
   ./stop_dashboard.sh
```

### 場景 3: 手機訪問

```
1. 確保手機和電腦在同一個WiFi

2. 啟動平台後，記下區網IP
   例如: http://192.168.1.100:5000

3. 在手機瀏覽器開啟該網址

4. 即可在手機上操作監控系統
```

## 🎨 Web 介面說明

### 頂部導航欄（藍色）
- 左側：AI主力監控平台標題 + 當前時間
- 右側：系統狀態指示燈
  - 🟢 綠色 = 運行中
  - 🔴 紅色 = 已停止

### 統計卡片（4個）
1. **系統狀態** - 顯示監控系統是否運行
2. **今日警示** - 今天發出的警示數量
3. **監控股票** - 監控清單中的股票數量
4. **交易時間** - 目前是否為交易時間

### 控制面板
- 大按鈕設計，易於操作
- 顏色標示：綠(啟動)、紅(停止)、藍(測試)、灰(刷新)

### 監控清單
- 顯示所有監控中的股票
- 每個股票後面有 ❌ 按鈕可快速移除

### 警示列表
- 最新的警示在最上面
- 左側彩色邊框標示信心度
  - 🔴 紅色 = ≥80% 高信心
  - 🟡 黃色 = 60-79% 中信心
  - 🟢 綠色 = <60% 低信心

## 🔧 進階功能

### 同時運行監控系統和 Web 平台

```bash
# 方案A: 使用兩個終端機
# 終端1
./start_monitor.sh  # 選擇2 (背景運行)

# 終端2
./start_dashboard.sh

# 方案B: 全部背景運行
./start_monitor.sh  # 選擇2
./start_dashboard_daemon.sh
```

### 查看 Web 平台日誌

```bash
# 實時查看
tail -f logs/dashboard.log

# 查看最後 50 行
tail -50 logs/dashboard.log
```

### 從其他裝置訪問

1. **查詢電腦 IP 地址**
   ```bash
   # macOS
   ifconfig | grep "inet " | grep -v 127.0.0.1
   
   # 或使用
   hostname -I
   ```

2. **在其他裝置開啟**
   ```
   http://[您的IP]:5000
   例如: http://192.168.1.100:5000
   ```

## 🐛 故障排除

### Q: 無法訪問 Web 平台？

**A**: 檢查：
```bash
# 1. 確認平台是否運行
ps aux | grep dashboard.py

# 2. 檢查端口是否被占用
lsof -i :5000

# 3. 重新啟動
./stop_dashboard.sh
./start_dashboard.sh
```

### Q: 點擊「啟動」無反應？

**A**: 
1. 檢查監控系統檔案是否存在
2. 確認有執行權限
3. 查看 Web 平台日誌
   ```bash
   tail logs/dashboard.log
   ```

### Q: 顯示「系統已停止」但實際在運行？

**A**: 刷新頁面或點擊「刷新」按鈕

###  Q: 添加股票失敗？

**A**: 
1. 確認股票代號格式正確（例如：2330.TW）
2. 檢查 config.yaml 檔案權限
3. 查看是否已存在於清單中
