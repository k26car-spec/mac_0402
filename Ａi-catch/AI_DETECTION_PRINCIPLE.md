# 🔍 AI 主力大單偵測原理說明

## 📊 核心概念

系統使用 **多維度特徵分析 + 加權評分** 的方法來偵測主力進場，而不是依賴單一指標。這種方法更穩健、更準確。

---

## 🎯 8 大核心特徵

### 1️⃣ **量能異常檢測** (Volume Anomaly)
**權重: 25%**

**原理:**
- 使用統計學的四分位數 (IQR) 方法
- 計算過去 20 天的成交量分佈
- 當今日成交量超過「上界」(Q75 + 1.5×IQR)，判定為異常放大

**判斷邏輯:**
```python
當前成交量 / 正常上限 > 1.5  ➜ 可能有主力進場
```

**實際案例 (5521 工信):**
- 正常成交量: 500-1000 張
- 12/15 當天: 1800 張 (超過正常值 1.8 倍)
- 量能異常分數: **0.9/1.0** ✅

---

### 2️⃣ **大單分析** (Large Order Ratio)
**權重: 30% (最重要!)**

**原理:**
- 分析成交量中「大單」的佔比
- 大單定義: 超過平均成交量 5 倍的交易
- 大單比例越高，主力色彩越濃厚

**判斷邏輯:**
```python
大單筆數 / 總筆數 > 0.3  ➜ 主力在操作
當前成交量 / (平均量 × 3) > 1.0  ➜ 大單進場
```

**為什麼重要？**
主力單筆下單量通常遠大於散戶，大單集中出現是主力最明顯的特徵

---

### 3️⃣ **資金流向指標** (Money Flow Index, MFI)
**權重: 15%**

**原理:**
- 類似 RSI，但結合了成交量
- 計算 14 天內「價漲量增」vs「價跌量增」的資金流向
- MFI > 70 表示強勁買盤湧入

**計算公式:**
```
典型價格 = (最高價 + 最低價 + 收盤價) / 3
資金流 = 典型價格 × 成交量
正資金流 = 價格上漲時的資金流總和
負資金流 = 價格下跌時的資金流總和
MFI = 100 - [100 / (1 + 正資金流/負資金流)]
```

**判斷:**
- MFI > 70: 強力買盤
- MFI > 80: 極強買盤 (主力可能性高)

---

### 4️⃣ **法人追蹤指標** (Institutional Flow)
**權重: 20%**

**原理:**
- 分析「價格變化」與「成交量變化」的相關性
- 法人買進特徵: 價漲量增 (正相關)
- 散戶恐慌特徵: 價跌量增 (負相關)

**計算方法:**
```python
相關係數 = corr(價格變化%, 成交量變化%)
```

**判斷:**
- 相關性 > 0.5: 法人持續買進
- 相關性 > 0.7: 法人大舉進場 ✅

---

### 5️⃣ **型態突破識別** (Pattern Breakout)
**權重: 10%**

**原理:**
- 計算股價與 20 日均線的乖離率
- 突破均線 + 量能配合 = 主力啟動訊號

**計算:**
```python
突破強度 = (當前價 - MA20) / MA20
```

**判斷:**
- 突破 > 3%: 短期強勢
- 突破 > 5%: 主力可能啟動

---

### 6️⃣ **價格動能** (Price Momentum)

**原理:**
- 計算 5 日價格漲跌幅
- 快速上漲通常伴隨主力推升

**公式:**
```python
動能 = (今日收盤 - 5日前收盤) / 5日前收盤
```

---

### 7️⃣ **委買委賣分析** (Bid-Ask Spread)

**原理:**
- 分析五檔委買委賣的力道對比
- 委買量 >> 委賣量 = 買盤強勁

**判斷:**
```python
委買量 / 委賣量 > 1.5  ➜ 買盤占優勢
```

---

### 8️⃣ **統計特徵** (Statistical Features)

**包含:**
- **偏度 (Skewness)**: 成交量分佈的偏斜程度
- **峰度 (Kurtosis)**: 價格波動的尖銳程度

**用途:** 補充判斷，識別異常交易模式

## 🧮 信心分數計算

### 加權平均公式

```python
信心分數 = Σ (特徵值(標準化) × 權重)

= 量能異常 × 0.25
+ 大單比例 × 0.30    ← 最重要
+ 資金流向 × 0.15
+ 法人追蹤 × 0.20
+ 型態突破 × 0.10
```

### 標準化方法

將不同量級的特徵統一轉換到 [0, 1] 區間:

| 特徵 | 標準化方法 |
|------|-----------|
| 量能異常 | `min(value / 2.0, 1.0)` |
| 大單比例 | `min(value / 0.5, 1.0)` |
| 資金流向 MFI | `value / 100` |
| 法人追蹤 | `(value + 1) / 2` (從 [-1,1] 轉到 [0,1]) |
| 型態突破 | `min(abs(value) / 0.1, 1.0)` |

## 💡 為什麼這個方法有效？

### 優勢

1. **多維度交叉驗證** - 不依賴單一指標，降低假訊號
2. **量化分析** - 客觀數據，不受情緒影響
3. **即時監控** - 每 60 秒檢查一次，捕捉進場時機
4. **可調整** - 可根據市場狀況調整權重和閾值

### 局限性

1. **需要足夠數據** - 至少需要 20 天歷史數據
2. **無法100%確定** - 只能識別「高可能性」
3. **延遲性** - 是「事後確認」而非「預測」
4. **數據源限制** - Yahoo Finance 不提供逐筆數據
