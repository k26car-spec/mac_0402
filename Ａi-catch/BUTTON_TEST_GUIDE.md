## 🧪 測試步驟

### 1. 刷新網頁

```
按 F5 或 Cmd+R 刷新頁面
```

### 2. 打開開發者工具

```
按 F12 或 Cmd+Option+I
點擊「Console」標籤
```

### 3. 點擊「啟動」按鈕

您應該看到：

#### 確認對話框：
```
確定要啟動監控系統嗎？

系統將開始監控股票並偵測主力進場。

[取消] [確定]
```

#### Console 輸出：
```
啟動按鈕被點擊
準備啟動監控系統...
發送 POST 請求到 /api/monitor/start
收到響應: 200
響應數據: {success: true, message: "監控已啟動"}
```

#### 成功提示：
```
✅ 監控已啟動

系統已開始監控。

[確定]
```

## 🐛 如果還是沒反應

### 檢查1: 查看 Console

按 F12，在 Console 中查看：

**有反應**：
```
啟動按鈕被點擊
用戶取消啟動
```
表示點擊有效，只是您按了「取消」

**完全沒反應**：可能是 JavaScript 錯誤

**有錯誤訊息**：複製錯誤信息

---

### 檢查2: 確認頁面完整載入

```javascript
// 在 Console 輸入：
typeof startMonitor
```

應該看到：
```
"function"
```

如果看到 `undefined`，表示頁面沒有完整載入。

---

### 檢查3: 手動測試 API

在 Console 輸入：

```javascript
fetch('/api/monitor/start', {method: 'POST'})
  .then(r => r.json())
  .then(d => console.log(d))
```

應該看到：
```
{success: true, message: "監控已啟動"}
```

## 💡 常見問題

### Q: 點擊後沒有確認對話框？

**A**: 可能瀏覽器阻擋了 JavaScript
- 檢查網址列是否有 🛡️ 圖示
- 允許此站點執行 JavaScript

### Q: 出現確認，但點確定後沒反應？

**A**: 
1. 查看 Console 的錯誤訊息
2. 確認 dashboard.py 在運行
3. 測試 API（見上方檢查3）

### Q: 顯示「監控已在運行中」？

**A**: 
- 這是正常的！表示系統已經啟動了
- 點擊「停止」後再點「啟動」
