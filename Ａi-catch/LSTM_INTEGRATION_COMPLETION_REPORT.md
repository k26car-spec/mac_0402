### Day 1 (2月10日)：問題診斷與突破

#### 上午 (11:20-13:30)
**任務**：
- 創建回測系統 (`lstm_backtest.py`)
- 執行 3 個月與 6 個月回測

**發現的問題** ⚠️：
- 3 個月回測：平均準確率 52.91%（接近目標）
- **6 個月回測：平均準確率僅 37.84%** ❌
- 精確率/召回率 = 0%（模型基本無效）

#### 下午 (13:30-21:00)
**關鍵突破** 💡：
- 診斷 **3 支代表性股票**：
  - 2314 (最佳, 72.73%)
  - 5521 (中等, 51.52%)
  - 2408 (最差, 6.06%)

**根本原因發現**：
```
問題：使用固定閾值 0.5 判斷漲跌
真相：每支股票的歸一化範圍不同
- 2314: 預測值 -0.045 (全是負數) < 0.5 → 全判「下跌」→ 碰巧對 (72.73%)
- 2408: 預測值 0.120 (正數) < 0.5 → 全判「下跌」→ 完全錯 (6.06%)
```

**解決方案**：為每支股票找專屬的「最佳判斷閾值」

**成果**：
- ✅ 創建診斷工具 (`diagnose_lstm_predictions.py`)
- ✅ 100% 確認問題根源
- ✅ 制定修復計畫

**架構設計**：
```
LSTMManager (AI 管理器)
├─ 讀取白名單 (lstm_whitelist.json)
├─ 懶加載模型 (需要時才載入)
├─ 數據預處理 (歷史 K 線 → LSTM 輸入)
└─ 提供預測接口 (signal + confidence)

Smart Entry System (交易系統)
├─ 技術指標分析 (原有功能)
├─ 🆕 調用 LSTM Manager
├─ 🆕 融合 AI 信號
│   ├─ AI 看漲 → +10~15 分
│   └─ AI 看跌 → -10~20 分
└─ 最終決策 (技術面 + AI 雙重驗證)
```

**集成邏輯**：
```python
# 在 calculate_confidence 方法中
lstm_result = lstm_manager.predict(stock_code, hist_df)

if lstm_result['signal'] == 1:  # AI 看漲
    confidence += 10~15  # 加分
    factors.append("🤖 AI推薦")
else:  # AI 看跌
    confidence -= 10~20  # 扣分
    factors.append("⚠️ AI預警")
```

## ⚠️ 注意事項與風險

### 系統限制

1. **數據依賴**：
   - LSTM 需要至少 60 根 K 線（約 3 個月數據）
   - 新上市股票無法立即使用

2. **準確率天花板**：
   - 平均 55% 意味著 45% 會出錯
   - **不應完全依賴 AI**，技術面仍是主導

3. **黑天鵝事件**：
   - 突發新聞、政策變動無法預測
   - 建議保留人工最終審核

### 使用建議

1. **保守策略** ⭐（推薦）：
   - AI 作為「過濾器」（只用來阻擋風險）
   - 技術面通過 + AI 不反對 → 進場

2. **平衡策略**：
   - AI 與技術面各佔 50%
   - 雙方共識才進場

3. **激進策略** ⚠️（不建議）：
   - 相信 AI 多過技術面
   - 風險較高

## 🎓 技術總結

### 核心技術

1. **LSTM (Long Short-Term Memory)**：
   - 深度學習中的時間序列模型
   - 擅長捕捉長期依賴關係
   - 適合股價預測

2. **自適應閾值 (Adaptive Threshold)**：
   - 解決歸一化範圍不一致問題
   - 為每支股票量身定做判斷標準

3. **類別權重平衡 (Class Weights)**：
   - 解決樣本不平衡問題
   - 強迫模型學習稀有事件

4. **懶加載 (Lazy Loading)**：
   - 節省記憶體
   - 提升啟動速度

### 創新點

1. **診斷式開發**：
   - 不盲目優化，先找根本原因
   - 3 支股票診斷 → 100% 驗證假設

2. **拯救行動設計**：
   - 不放棄任何股票
   - 18/18 全救回（100% 成功率）

3. **漸進式集成**：
   - 先建管理器，再修改主系統
   - 錯誤容錯設計（AI 故障不影響主流程）

## 📞 技術支援

### 常見問題

**Q1: 如果某支股票的 LSTM 不在白名單，會發生什麼？**
A: 系統會自動跳過 LSTM，純用技術指標判斷，不影響正常運作。

**Q2: 如何新增股票到白名單？**
A: 
1. 訓練該股票的 LSTM 模型
2. 用 `lstm_backtest_with_adaptive_threshold.py` 回測
3. 如果準確率 ≥50%，手動加入 `lstm_whitelist.json`

**Q3: AI 判斷錯誤怎麼辦？**
A: 
- AI 只是輔助，最終決策仍由綜合分數決定
- 定期追蹤 AI 命中率，如果持續 <45% 就淘汰該股票模型

**Q4: 系統變慢怎麼辦？**
A: 
- LSTM 預測需要網絡抓取歷史數據，可能較慢
- 優化方向：建立本地數據庫，避免每次都抓取
